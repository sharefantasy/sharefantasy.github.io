Server Consolidation with Minimal SLA Violations

[TOC]

作者信息

2016 Computational Intelligence In Data Mining, Cidm, Vol 2
Patel, C. A.　 Shah, J. S.

提出的约束条件

术语定义

$ P = {P1 , P_2 ,….., P_m} $is set of m physical machines
$ P_i^C = $ Total Capacity of $ P_M \; P_i $ in terms of tuple (RAM, CPU, N/W)
$ P_i^U = $ Utilized Capacity of $ P_M \; P_i $ in terms of tuple (RAM, CPU, N/W)
$ V = {V_1, V_2,….., V_n} $ set of n virtual machines in data center
$ V_j^R = $ required capacity of $
V_M \; V_i $ terms of tuple (RAM, CPU, N/W)
$ M = m × n $ matrix where
$ M{ij} = \begin{cases} 1& if\;VM V_j \text{ is assigned on }P_M \; P_i\ 0& \text{otherwise} \end{cases} $
$ S = {S_1, S_2,….. S_s} $ is a set of SLAs
$ th = $ Threshold defined by user e.g. 80 % (to identify over utilization of host)
$ mig{th} = $ Threshold defined by user e.g. 3 (to specify maximum number of allowed migrations for each VM)

约束方程

Use minimum number of active PMs

$Minimize \; \sum{i=1}^{m}{Y_i} $
$Y_i = \begin{cases} 1 & \sum{j=1}^{n}{M_{ij}} \ge 1 \ 0 & \text{Otherwise} \end{cases} $

$\text{Constrained by}$

$ \text{C1: } \sum{i=1}^{m}M{ij} = 1, \; \forall \, Vj \in VM $
$ \text{C2: } migrationCount(V_j ) \le mig{th}, \; \forall \, V_j \in VM $

$\sum_{i=1}^{m}{Y_i}$ represents total number of active servers.
Each $Y_i$ is one, if number of hosted VMs on that server is greater than or equal to 1.
(最小活跃物理服务器)

Constraint C1 ensures that VM cannot be hosted partially on two different servers.
C2 ensures that same performance of same VM is not degraded all time.
(C1保证每个VM整体放置于同一服务器，C2保证同一VM的迁移次数不高于阈值)

算法实现

Schedule_VM (Fist fit based approach)

Optimize_DC (MMT as selection criteria)

实验

CloudSim simulator is used to perform experiments. Three parameters were considered in experiments namely:

  1. energy consumption of datacenter
  2. SLA performance degradation of VMs
  3. SLA time per active host.

Experiment scenario is as bellow:

  • VM allocation Policy: Best Fit decreasing
  • VM selection Policy: Minimum migration time
  • Overload threshold th=0.8
  • Max migration per VM mig_th=3
  • Workload: Planetlab
