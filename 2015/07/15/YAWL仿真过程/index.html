YAWL仿真过程

发表于 2015-07-15   |   分类于 研究笔记   |  

工具

  • 必须：
    1. ProM 5.2(6.3功能不明，暂不使用）
    2. CPN Tools
    3. ProM import framework（简称PIF）

  • 可选：

    1. YAWL（设计时用）

    YAWL仿真工具架构上分为3个部分，分别是YAWL，ProM和CPN Tools.
    YAWL用于设计、定义流程模型和组织模型，和记录工作流引擎目前状态；
    ProM把yawl net翻译成petri net，并使用历史数据或者仿真数据进行流程挖掘、分析。
    CPN Tools导入CPN并进行仿真，生成仿真记录。

总体流程

  1. 准备好流程模型、组织模型
  2. 使用ProM的Merge Simulation Models把前两者合并，并转化为HLpetrinet
  3. 使用ProM的Export to CPN Tools 2.0将HLpetrinet转化成CPN，并设置各仿真参数
  4. 使用CPN Tools打开CPN文件，设定case数，跑仿真
  5. 使用PIF把仿真记录转换成mxml文件，使用ProM打开并做分析挖掘。
  6. 调整模型

具体流程

  1. 准备流程模型和组织模型。
    注意两者扩展名都是xml.不是导出时的。 yawl specification可以直接改扩展名，载入时选择 newYAWLfile加载就可以，组织模型还没明白。

  2. 将两者加载到ProM里，选择analysis中的Merge Simulation Models，调整参数后生成Merge net。

  3. 选择Conversion中的HLyawlnet to HLpetrinet，生成

  4. 选择Analysis中的Export to CPN Tools 2.0，调整好参数，保存。(重点)
    暂时还不知道其他选项用处，最好全部勾上。
    不要勾选有waiting time的，加载到CPN Tools时有错误，似乎是HLyawlnet转成HLpetrinet过程中会生成新的弧，那些弧缺乏定义。

  5. 使用CPN Tools打开生成的cpn文件，等待syntax check，可能需要数分钟。在检查通过（文件名下划线变成绿色其他全部消失）之前，无法做仿真。
    *注：关于CPN tools的syntax check可以参考 此处，当检查不通过时需要调整export时的参数。

  6. 打开simulation面板，选择一种仿真工具进行仿真。
    这些工具都是以transition为单位推进的，可以通过右键点击设置工具选项来调整仿真次数（步数、时长等）

  7. 仿真完毕后，用PIF设置好导入目录和输出目录，将cpn的日志文件转化为mxml文件。
    红框两个必须设置，黄框的最好设置成截图那样。

  8. 把mxml文件导入到ProM中，就能愉快地做下一步分析了。

问题

  1. 官方教程提到将模型、组织模型、日志文件3者合并在一起再转化，事实上只用前2个。到底是怎样？
  2. ProM不能加载ybkp文件，那么组织模型那个格式是怎么来的？
  3. 转成HLpetrinet后，似乎丢失了一些弧的信息，使得CPN不能进行带waiting time的仿真。该怎么处理？
  4. 只能靠分析日志，然后人手调整参数吗？好像有点落后
