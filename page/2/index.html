模型定义与算法设计

发表于 2016-01-26   |  

一、实验环境
1.环境
工程环境：代码实际运行场景中的环境，具有不确定性和模糊性，可能掺杂其他不确定因素
模型环境：抽象后的运行环境，只关注建模重点，忽略细节。难以描述但有重要的地方使用概率密度函数描述
时槽： 所有环境、函数中通用的时刻标记，有固定间隔， t ∈ N+

  1. 时间单位
    总周期： 集群开始运行到结束的总时间
    结算周期：租户合约的结算单位周期，所有统计数据的总时间均为一个结算周期，结算周期过后所有数据清零。
    操作周期：集群调整空间配置的时间，每个操作周期判断并实行负载均衡
    心跳周期：集群收集统计数据的单位时长

二、引擎
1、操作定义
请求： 环境中要求一次工作流服务的过程
派发： 接受一个请求生成workitem并指定worklist的过程
引擎： 工程环境中运行的代码，产生派发的工具，可以是工作流引擎，或封装了工作流引擎的VM/容器，模型环境中则是（宿主参数，自身状态->派发速度）函数，生成引擎需要考虑以下几个因素：（有无排队机制，有无速度限制，有无速度负反馈机制）
宿主： 引擎所在的物理机

2.指标定义
请求速度： 单个时槽中的请求数量
请求总数： 整个时槽轴上的请求数量
派发速度： 单个时槽中的派发数量
派发总数： 整个时槽轴上的派发数量

二、租户
1.限制
所有租户都在总周期的0时刻上线并统一在总周期结束后下线，不考虑中途插入，不考虑多层SLA，不考虑SLA renegotiation。违约租户惩罚统一，引擎迁移成本不计，分配到租户的引擎数为先验计算，不可增减。
可能改进：租户插入/早退，更改，惩罚成本差异，迁移成本计入模型。

  1. 指标定义
    违约时长：结算周期中，不满足租户SLO速度的心跳时槽之和
    单人违约率 = 违约时长 / 结算周期

3.SLA定义
SLO速度：结算周期中，约定的最低引擎派发速度
最高故障比例：单人违约率的约定最大值

4.运维过程
已违约租户 = 结算周期内，单人违约率已超过最高故障比例的租户
集群违约比例 = 已违约租户数 / 总租户数

云工作流引擎设计感想与计划

发表于 2016-01-11   |  

一、概念简介

　　工作流：业务过程的部分或整体在计算机应用环境下的自动化。
　　云计算：基于互联网的相关服务的增加、使用和交付模式，通常涉及通过互联网来提供动态易扩展且经常是虚拟化的资源。另外云计算是一种按使用量付费的模式，这种模式提供可用的、便捷的、按需的网络访问，进入可配置的计算资源共享池（资源包括网络、服务器、存储、应用软件、服务），这些资源能够被快速提供，只需投入很少的管理工作，或与服务供应商进行很少的交互。
　　云工作流：按照云计算的定义，根据工作流引擎的性能、资源使用量计费，提供多租户、有分级的工作流引擎的PaaS服务。通常铺设在云或网格平台上，以分布式引擎的方式部署提高并行化以及高可用性保障，并以虚拟化技术和接口技术模拟传统集中式业务引擎，另外通过工作流性能统计、预测租户对各种资源的使用量，进行计费。云工作流目的不在于使用分布式系统集中提高性能，而是为用户分摊维护成本。另外，云工作流也有另一种定义，被称为“云之上”工作流，即研究如何通过组合云环境中的服务，构建合理高效可靠的业务流程实践。相对应的本项目被称为“云之中”工作流。“云之上”并非本项目的研究方向。

YAWL仿真过程

发表于 2015-07-15   |   分类于 研究笔记   |  

工具

  • 必须：
    1. ProM 5.2(6.3功能不明，暂不使用）
    2. CPN Tools
    3. ProM import framework（简称PIF）

  • 可选：

    1. YAWL（设计时用）

    YAWL仿真工具架构上分为3个部分，分别是YAWL，ProM和CPN Tools.
    YAWL用于设计、定义流程模型和组织模型，和记录工作流引擎目前状态；
    ProM把yawl net翻译成petri net，并使用历史数据或者仿真数据进行流程挖掘、分析。
    CPN Tools导入CPN并进行仿真，生成仿真记录。

总体流程

  1. 准备好流程模型、组织模型
  2. 使用ProM的Merge Simulation Models把前两者合并，并转化为HLpetrinet
  3. 使用ProM的Export to CPN Tools 2.0将HLpetrinet转化成CPN，并设置各仿真参数
  4. 使用CPN Tools打开CPN文件，设定case数，跑仿真
  5. 使用PIF把仿真记录转换成mxml文件，使用ProM打开并做分析挖掘。
  6. 调整模型
